# 바이브코딩(vibe coding) 1차 GPT(Generative Pre-trained Transformer) 입문서 원고 작업 단위표

## 작업 단위 원칙

- Writer 1회 작업은 한 섹션 또는 인접한 짧은 섹션 묶음으로 제한합니다.
- 각 단위는 목적 하나만 가집니다.
- 원고에는 독자가 직접 파일을 만들거나 터미널을 쓰는 안내를 넣지 않습니다.
- Chapter 2와 Chapter 3의 실습 단위는 설명, 프롬프트(prompt) 핵심 3줄, 복사하기 전문, ChatGPT 입력 캡처(capture), HTML(HyperText Markup Language) 결과 이미지 구조를 따릅니다.
- 원고 본문 작성 전에는 필요한 캡처(capture) 번호와 검수 기준을 먼저 연결합니다.
- Chapter 1은 개념 도입 장이며 결과물 이미지를 넣지 않습니다.
- 같은 이미지 파일은 책 전체에서 한 번만 사용합니다.

## 역할별 작업 기준

| 역할 | 책임 | 이번 구조에서 볼 것 |
|---|---|---|
| Writer | 한 단위씩 본문 작성 | 설명, 프롬프트(prompt), 이미지 흐름 유지 |
| Editor | 문장과 리듬 정리 | 장황한 설명 압축 |
| Reviewer | 일반 독자 시점 검수 | 지금 무엇을 하면 되는지 확인 |
| Verifier | 기준 위반 검수 | 금지 기준, 캡처(capture), 배포 URL(Uniform Resource Locator) 확인 |
| Capture | 실제 화면 캡처(capture) | Playwright 입력 캡처(capture)와 HTML(HyperText Markup Language) 결과 캡처(capture) 구분 |

# Chapter 1 이미지 제작계획

## 문서 목적

이 문서는 Chapter 1 “AI(Artificial Intelligence), LLM(Large Language Model), 바이브코딩(vibe coding) 개요와 시장 소개”의 이미지 정책을 고정합니다.

## 확정 기준

- Chapter 1에는 이미지 파일을 새로 만들지 않습니다.
- Chapter 1에는 첫 웹사이트 결과와 타로(tarot) 사이트 결과를 넣지 않습니다.
- 첫 웹사이트 결과 이미지는 Chapter 2에서만 사용합니다.
- 타로(tarot) 사이트 결과 이미지는 Chapter 3에서만 사용합니다.
- 같은 이미지 파일은 책 전체에서 한 번만 사용합니다.
- Chapter 1은 짧은 문단과 Chapter 2로 넘어가는 연결 문장으로 역할을 끝냅니다.

## 변경 이유

기존 계획은 Chapter 1에서 결과물 이미지를 먼저 보여준 뒤 Chapter 2와 Chapter 3에서 같은 성격의 이미지를 다시 보여주는 구조였습니다. 이 방식은 중복이 많고, 독자가 실제 실습 장면에서 보아야 할 결과 이미지를 앞에서 소모하게 됩니다.

수정 후 Chapter 1은 개념 도입만 담당합니다. 결과물 이미지는 실제 제작 흐름이 있는 Chapter 2와 Chapter 3에 배치합니다.

## 제작하지 않는 이미지

- 표지용 주제 이미지
- AI(Artificial Intelligence) 생활 도구 개념 이미지
- LLM(Large Language Model) 개념 이미지
- ChatGPT 대화 흐름 개요 이미지
- 바이브코딩(vibe coding) 흐름 다이어그램
- 일반 코딩과 바이브코딩(vibe coding) 비교 이미지
- 이미지 70%, 글 30% 책 구조 예시
- 첫 웹사이트 결과 선공개 이미지
- 타로(tarot) 사이트 결과 선공개 이미지
- 완성 결과물 요약 이미지

## 다음 기준

Chapter 2와 Chapter 3의 이미지 목록은 `이미지_캡처(capture)_목록.md`를 기준으로 관리합니다. 실제 HTML(HyperText Markup Language) 확인본에서는 이미지 경로 중복이 없어야 합니다.

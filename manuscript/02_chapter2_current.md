# Chapter 2. 이미지 하나로 웹사이트 만들고 배포합니다

## 이미지 하나로 첫 웹사이트를 만듭니다

이번 장에서는 이미지 하나를 ChatGPT에 넣고, 그 이미지와 어울리는 첫 웹사이트를 만듭니다.

그다음 ChatGPT와 GitHub를 연결해 결과를 웹 주소로 열 수 있게 배포합니다. 이 장을 끝내면 여러분은 “대화로 만든 결과가 실제 주소로 열리는 경험”을 하게 됩니다.

![이미지 자리: 첫 사이트에 사용할 이미지 예시](../webbook/images/ch02_02_select_image.png)

이 장의 흐름은 단순합니다.

- 이미지 한 장을 준비합니다.
- ChatGPT에 이미지를 첨부합니다.
- 프롬프트를 보내 첫 웹사이트를 만듭니다.
- 마음에 들지 않는 부분은 말로 고칩니다.
- GitHub Pages로 배포해 주소로 열어 봅니다.

## 시작 전에 준비합니다

이번 장은 책을 읽기만 하는 장이 아니라, 독자가 직접 ChatGPT에 같은 요청을 보내 보는 장입니다.

먼저 ChatGPT에 로그인되어 있는지 확인합니다. 이미지 첨부와 GitHub 연결은 로그인 상태에서 진행해야 안정적입니다.

GitHub도 구글 계정으로 로그인해 둡니다. 책에서는 구글 계정 기준으로 화면을 설명합니다.

첫 웹사이트에 사용할 이미지는 이 책의 버블 공연 사진을 기준으로 진행합니다. 다른 이미지를 쓰면 결과 화면의 분위기와 구성은 달라질 수 있습니다.

책에 실린 화면과 최대한 비슷하게 만들고 싶다면, 프롬프트를 직접 줄여 쓰지 말고 복사하기 버튼으로 전문을 붙여 넣습니다.

## ChatGPT에 이미지를 첨부합니다

ChatGPT 입력창 옆의 첨부 단추를 눌러 이미지를 올립니다. 이미지가 올라오면 입력창 근처에 작은 미리보기가 보입니다.

![이미지 자리: ChatGPT 이미지 첨부 직전 화면](../webbook/images/ch02_03_attach_image.png)

개인정보가 보이는 화면은 출판용 캡처에서 가립니다. 여러분이 직접 따라 할 때는 본인의 계정 화면이 보여도 괜찮습니다.

## 첫 프롬프트를 보냅니다

이제 ChatGPT에게 이미지를 바탕으로 웹사이트를 만들어 달라고 부탁합니다.

> 프롬프트 박스: first-site
> 표시: 앞 3줄 미리보기
> 버튼: 복사하기

복사하기 버튼을 누른 뒤 ChatGPT 입력창에 붙여 넣고 보내 주세요.

## 결과가 나올 때까지 기다립니다

ChatGPT가 결과를 만드는 동안 잠시 기다립니다. 답변이 길게 나오더라도 정상입니다.

답변이 끝나면 ChatGPT가 만든 결과를 확인합니다. 코드처럼 보이는 내용이 나와도 직접 외우거나 고칠 필요는 없습니다. 이 책에서는 최종 확인을 ChatGPT 대화창 안이 아니라 실제 브라우저 화면에서 합니다.

## 실제 브라우저 결과를 확인합니다

첫 결과 화면이 보이면 성공입니다.

아직 디자인이 마음에 꼭 들지 않아도 괜찮습니다. 다음 단계에서 말로 고치면 됩니다.

결과 예시는 아래 주소에서도 확인할 수 있습니다.

## 한 줄로 수정 요청을 보냅니다

수정은 어렵게 말하지 않아도 됩니다.

“글자를 조금 크게 해 주세요”, “배경을 더 밝게 해 주세요”처럼 한 줄로 부탁하면 됩니다.

> 프롬프트 박스: first-site-revise
> 표시: 앞 3줄 미리보기
> 버튼: 복사하기

수정 요청 뒤에는 결과가 어떻게 달라졌는지 봅니다.
수정 전과 크게 달라 보이지 않는다면 억지로 다시 캡처를 넣지 않습니다. 이 책에서는 같은 장면이 반복되지 않도록, 수정 요청을 보내는 방법만 익히고 다음 단계로 넘어갑니다.

## GitHub와 ChatGPT를 연결합니다

이제 첫 번째 사이트를 웹 주소로 열 수 있게 준비합니다.

GitHub는 만든 파일을 올려 두고 웹주소로 열 수 있게 도와주는 공간입니다. 이 책에서는 GitHub 화면을 깊게 배우기보다, ChatGPT와 연결해 대화로 배포하는 흐름을 사용합니다.

로그인은 구글 계정 기준으로 설명합니다. 이미 구글 계정으로 ChatGPT와 GitHub에 로그인되어 있다면 같은 흐름으로 확인하면 됩니다.

![이미지 자리: GitHub 메인 화면 전체](../webbook/images/ch03_02_github_home.png)

ChatGPT에서 GitHub 연결 위치를 찾습니다.

![이미지 자리: ChatGPT 설정 또는 커넥터 진입 화면](../webbook/images/ch03_06_chatgpt_connector_entry.png)

GitHub 연결을 선택하고 권한 화면이 나오면 내용을 확인한 뒤 승인합니다. 이 과정은 계정 상태에 따라 화면이 조금 다를 수 있습니다.

## 배포를 부탁합니다

연결이 끝나면 ChatGPT에게 첫 사이트를 GitHub Pages로 올려 달라고 부탁합니다.

> 프롬프트 박스: github-deploy-first
> 표시: 앞 3줄 미리보기
> 버튼: 복사하기

배포 과정은 시간이 조금 걸릴 수 있습니다. ChatGPT가 저장소를 만들고 파일을 올리고 Pages 설정을 안내하거나 진행합니다.

권한 화면이 나오면 내용을 확인한 뒤 승인합니다. 연결 화면이 나오지 않거나 ChatGPT가 직접 진행할 수 없다고 답하면, 부록의 “GitHub Pages 주소가 안 뜰 때” 문장을 사용해 현재 화면을 설명합니다.

## 배포 URL을 확인합니다

성공하면 ChatGPT 답변 안에 웹주소가 보입니다. 보통 `https://...github.io/...` 형태의 주소입니다.

![이미지 자리: ChatGPT 최종 주소 답변 화면](../webbook/images/ch03_12_final_url.png)

주소가 열리고 첫 번째 사이트가 보이면 배포 성공입니다.

![이미지 자리: 첫 사이트 데스크톱 외부 URL 결과 화면](../webbook/images/ch03_13_first_site_public_desktop.png)

## 이 장에서 확인할 것

- 이미지 한 장을 ChatGPT에 첨부했습니다.
- 첫 사이트 제작 프롬프트를 복사해 보냈습니다.
- 첫 웹사이트 결과 화면을 확인했습니다.
- 한 줄 수정 요청을 보내 봤습니다.
- GitHub와 ChatGPT 연결 흐름을 확인했습니다.
- 배포 URL로 첫 웹사이트가 열리는지 확인했습니다.

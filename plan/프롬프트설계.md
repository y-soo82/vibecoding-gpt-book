# 프롬프트 박스 UX 설계

## 목표

긴 프롬프트를 본문에 모두 펼치지 않습니다. 독자는 앞 3줄만 보고, 우측 상단의 `복사하기` 버튼으로 전문을 복사합니다.

## UX 원칙

- 프롬프트는 독자가 읽고 외우는 글이 아니라 실행하는 도구입니다.
- 본문 흐름은 이미지 중심으로 유지합니다.
- 프롬프트 전문은 복사 버튼과 전체 보기로 접근합니다.
- 모바일에서도 버튼이 손가락으로 누르기 쉬워야 합니다.
- EPUB에서는 JavaScript 복사가 제한될 수 있으므로 전문 보기 대체 경로를 제공합니다.

## 구성 요소

| 요소 | 역할 | 필수 |
|---|---|---|
| 제목 | 어떤 행동에 쓰는 프롬프트인지 알려 줍니다 | 필수 |
| 3줄 미리보기 | 본문 리듬을 유지합니다 | 필수 |
| 복사하기 버튼 | 전문을 클립보드에 넣습니다 | 필수 |
| 복사됨 피드백 | 사용자가 성공을 확인합니다 | 필수 |
| 전체 보기 토글 | 전문을 확인하고 싶은 독자를 배려합니다 | 권장 |
| 전문 보기 링크 | EPUB 대체 경로입니다 | 필수 |
| 실패 안내 | 복사가 안 될 때 직접 복사 방법을 알려 줍니다 | 권장 |

## 화면 배치

```text
┌─────────────────────────────────────┐
│ 첫 번째 사이트 만들기       복사하기 │
├─────────────────────────────────────┤
│ 이 이미지와 어울리는 반응형 웹사이트를... │
│ 휴대전화와 컴퓨터에서 보기 좋게 해 주세요. │
│ 따뜻한 분위기의 한 페이지로 구성해 주세요. │
│                                     │
│ 전체 보기                           │
└─────────────────────────────────────┘
```

## UI 규칙

- 미리보기는 앞 3줄만 표시합니다.
- 3줄을 넘는 내용은 흐림 처리하지 않고 잘라 보입니다.
- 복사하기 버튼은 우측 상단에 둡니다.
- 복사 성공 시 1.5초 동안 `복사됨`으로 바뀝니다.
- 복사 실패 시 `직접 복사` 안내 문장을 보여 줍니다.
- 전체 보기 버튼을 누르면 전문이 펼쳐집니다.
- 다시 누르면 3줄 미리보기 상태로 돌아갑니다.
- 버튼 높이는 모바일 기준 최소 40px입니다.
- 버튼이 프롬프트 문장을 가리지 않아야 합니다.
- 프롬프트 박스 안에서 가로 스크롤이 생기지 않아야 합니다.

```html
  <div class="prompt-card__header">

  <pre class="prompt-card__preview"></pre>
  <button class="prompt-card__toggle" type="button">전체 보기</button>
  <pre class="prompt-card__full" hidden></pre>

```

## 반응형 기준

| 화면 폭 | 기준 |
|---:|---|
| 390px | 버튼과 제목이 겹치지 않아야 합니다 |
| 430px | 3줄 미리보기가 안정적으로 보여야 합니다 |
| 768px | 프롬프트 박스 폭이 본문 폭을 넘지 않아야 합니다 |
| 1280px | 본문이 과하게 넓어지지 않아야 합니다 |

## EPUB 대체 전략

- 웹북에서는 복사 버튼을 제공합니다.
- EPUB에서는 리더에 따라 JavaScript가 제한될 수 있습니다.
- 각 프롬프트 뒤에는 `프롬프트 전문 보기` 링크를 둡니다.
- EPUB용 원문 페이지에는 프롬프트 전문을 일반 텍스트로 제공합니다.
- EPUB에서 복사 버튼이 작동하지 않아도 독자가 전문을 확인할 수 있어야 합니다.

## 검수 기준

- 모바일 390px에서 버튼이 프롬프트를 가리지 않습니다.
- 복사 버튼을 누르면 전문이 클립보드에 들어갑니다.
- 복사 후 `복사됨` 피드백이 보입니다.
- 전체 보기 버튼으로 전문을 펼칠 수 있습니다.
- 전문 보기 링크가 존재합니다.
- 프롬프트 원문과 본문 미리보기의 앞 3줄이 일치합니다.

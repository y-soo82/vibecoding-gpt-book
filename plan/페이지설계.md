# 페이지 설계

## 목표

- 전체 분량은 80~100페이지로 맞춥니다.
- 기획 단계 기준으로는 98페이지를 목표로 합니다.
- 이미지 비중은 전체 60~70%를 목표로 합니다.
- 한 페이지에는 하나의 행동 또는 하나의 판단만 담습니다.
- 긴 설명보다 화면 캡처, 번호, 짧은 캡션을 우선합니다.
- 프롬프트 전문은 본문 페이지를 과하게 차지하지 않도록 별도 원문과 복사 UX로 분리합니다.

## 페이지 배분

| 구간 | 목표 페이지 | 이미지 비중 | 핵심 역할 |
|---|---:|---:|---|
| 머리말 | 5 | 50% | 책 사용법과 복사 버튼 안내 |
| Chapter 0 | 7 | 55% | AI와 바이브코딩 시장 감각 |
| Chapter 1 | 9 | 70% | ChatGPT 대화창 기본 이해 |
| Chapter 2 | 11 | 70% | 이미지 한 장으로 첫 사이트 생성 |
| Chapter 3 | 15 | 75% | ChatGPT GitHub 연동 배포 |
| Chapter 4 | 7 | 70% | 타로 사이트 완성 모습 이해 |
| Chapter 5 | 8 | 65% | 타로 이미지 준비와 한도 회피 |
| Chapter 6 | 22 | 70% | 타로 사이트 제작 |
| Chapter 7 | 9 | 70% | 타로 사이트 배포 |
| 부록 | 5 | 45% | 막힘 대응 문장 |
| 합계 | 98 | 약 67% | 출판 기준 분량 |

## 압축 기준

98페이지를 넘기면 아래 순서로 줄입니다.

1. 부록의 유사한 재요청 문장을 합칩니다.
2. Chapter 6의 같은 화면 반복 캡처를 줄입니다.
3. Chapter 0의 시장 설명을 카드형 이미지 2장으로 압축합니다.
4. Chapter 7의 배포 복습 문장을 Chapter 3 참조로 줄입니다.

## 확장 기준

원고가 80페이지보다 부족하면 아래 순서로 보강합니다.

1. Chapter 6의 타로 사이트 완성도를 높입니다.
2. 모바일 결과 확인 컷을 보강합니다.
3. 오류 수정 프롬프트 예시를 부록에 추가합니다.
4. 세 번째 사이트는 본문 확장용이 아니라 부록 후보로만 둡니다.

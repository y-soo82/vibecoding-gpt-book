# 이미지 컷 리스트

## 공통 이미지 규칙

- 전체 책의 이미지 비중은 60~70%입니다.
- 한 행동에는 최소 1장 이상의 이미지가 필요합니다.
- 처음 보는 화면은 전체 화면 1장을 먼저 보여 줍니다.
- 눌러야 할 곳은 클로즈업 이미지로 다시 보여 줍니다.
- 번호는 캡처 위에 과하게 크게 넣지 않습니다.
- 이미지 번호는 실제 UI 글자보다 약 20% 작게 배치합니다.
- 번호는 버튼, 입력칸, 본문 글자를 가리지 않습니다.
- 같은 이미지를 반복하지 않습니다.
- 같은 화면을 다시 써야 하면 확대 위치, 표시 번호, 설명 목적을 다르게 합니다.
- 영어 UI가 나오는 화면에는 짧은 한국어 캡션을 붙입니다.
- 깨진 이미지, 빈 placeholder, TODO 문구가 보이면 출고하지 않습니다.

## 파일명 규칙

```text
ch{챕터번호}_{페이지번호}_{장면설명}.png
```

예시:

```text
ch03_08_github_authorize.png
ch06_14_tarot_result_desktop.png
ch06_20_tarot_mobile.png
```

## 이미지 검수 체크리스트

- 번호와 본문 단계 번호가 일치해야 합니다.
- 번호 크기는 실제 UI 글자보다 약 20% 작아야 합니다.
- 번호가 클릭할 버튼이나 입력칸을 가리면 안 됩니다.
- 모바일에서도 캡처 안 글자가 읽혀야 합니다.
- Chapter 3에는 GitHub 연동과 권한 승인 컷이 반드시 있어야 합니다.
- Chapter 6에는 데스크톱과 모바일 결과 컷이 모두 있어야 합니다.
- 같은 캡처를 반복 사용하면 안 됩니다.

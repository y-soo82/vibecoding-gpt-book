# 배포 확인 및 캐시 무효화 계획

## 목표

원격 환경에서는 로컬 미리보기만으로 완료 판단하지 않습니다. 최종 확인은 GitHub Pages 또는 외부 공개 URL에서 진행합니다.

## 확인 대상

| 대상 | 확인 방식 | 필수 |
|---|---|---|
| 반응형 웹북 | GitHub Pages 공개 URL | 필수 |
| 첫 번째 사이트 | GitHub Pages 공개 URL | 필수 |
| 타로 사이트 | GitHub Pages 공개 URL | 필수 |
| EPUB 준비물 | Sigil import 경로 확인 | 필수 |

## URL 버전 규칙

확인용 URL에는 항상 버전 쿼리를 붙입니다.

```text
https://example.github.io/vibecoding-gpt/?v=20260515-001
https://example.github.io/first-site/?v=20260515-001
https://example.github.io/tarot-site/?v=20260515-001
```

버전 값은 아래 형식을 씁니다.

```text
YYYYMMDD-빌드번호
```

예시:

```text
20260515-001
20260515-002
```

## 캐시 무효화 규칙

### HTML

`index.html`에는 캐시 방지 메타 태그를 넣습니다.

```html
<meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate">
<meta http-equiv="Pragma" content="no-cache">
<meta http-equiv="Expires" content="0">
```

### CSS와 JavaScript

CSS와 JavaScript에는 버전 쿼리를 붙입니다.

```html
<link rel="stylesheet" href="./assets/book.css?v=20260515-001">
<script src="./assets/prompts.js?v=20260515-001"></script>
<script src="./assets/book.js?v=20260515-001"></script>
```

### 이미지

이미지 경로에도 필요하면 버전 쿼리를 붙입니다.

```html
<img src="./images/ch03_08_github_authorize.png?v=20260515-001" alt="GitHub 권한 승인 화면">
```

### Docsify 사용 시

Docsify를 사용할 경우 noCache 설정을 사용합니다.

```html
<script>
  window.$docsify = {
    name: "바이브코딩 GPT 입문서",
    loadSidebar: true,
    noCache: true
  };
</script>
```

## 외부 확인 절차

1. GitHub Pages 배포 URL을 엽니다.
2. 확인용 버전 쿼리를 붙여 다시 엽니다.
3. 일반 브라우저에서 확인합니다.
4. 시크릿 창에서 확인합니다.
5. 모바일 화면 폭에서 확인합니다.
6. CSS와 JavaScript가 최신 버전으로 로드되는지 확인합니다.
7. 이미지가 최신 파일로 보이는지 확인합니다.
8. 프롬프트 복사 버튼이 보이는지 확인합니다.
9. 페이지 하단 또는 HTML 주석의 빌드 번호가 최신인지 확인합니다.

## 확인 뷰포트

| 화면 | 폭 | 목적 |
|---|---:|---|
| 모바일 소형 | 390px | 가장 좁은 독서 환경 확인 |
| 모바일 일반 | 430px | 일반 휴대전화 확인 |
| 태블릿 | 768px | 표와 이미지 배치 확인 |
| 데스크톱 | 1280px | 본문 폭과 이미지 가독성 확인 |

## 캐시 문제 판단 기준

아래 상황이 있으면 캐시 문제로 판단합니다.

- 수정한 문장이 외부 URL에서 보이지 않습니다.
- CSS 수정이 반영되지 않습니다.
- 이미지가 예전 화면으로 보입니다.
- 일반 창과 시크릿 창의 화면이 다릅니다.
- `?v=새버전`을 붙였을 때만 수정 내역이 보입니다.

## 캐시 문제 대응 순서

1. URL에 최신 버전 쿼리가 붙었는지 확인합니다.
2. CSS, JS, 이미지 경로의 버전 쿼리가 최신인지 확인합니다.
3. GitHub Pages 배포 시간이 충분히 지났는지 확인합니다.
4. 시크릿 창에서 같은 URL을 엽니다.
5. 모바일 네트워크 또는 다른 브라우저에서 확인합니다.
6. 그래도 안 보이면 빌드 번호를 바꾸고 다시 배포합니다.

## 빌드 번호 표기

웹북에는 확인용 빌드 번호를 남깁니다.

```html
<!-- build: 20260515-001 -->
```

화면 하단에는 독자에게 거슬리지 않는 작은 표기를 둘 수 있습니다.

```text
버전 20260515-001
```

## 완료 기준

- 외부 URL에서 웹북이 열립니다.
- 외부 URL에서 첫 번째 사이트가 열립니다.
- 외부 URL에서 타로 사이트가 열립니다.
- 일반 창과 시크릿 창에서 같은 최신 화면이 보입니다.
- 모바일 화면에서 이미지와 프롬프트 박스가 깨지지 않습니다.
- 최신 빌드 번호가 확인됩니다.
- `reports/publish-check.md`에 확인 결과가 기록됩니다.

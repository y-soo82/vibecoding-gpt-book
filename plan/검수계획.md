# Reviewer/Verifier 검수 계획

## 목적

이 검수 계획은 바이브코딩 1차 GPT 입문서가 비개발자 독자에게 실제로 따라 하기 쉬운지, 그리고 출판·배포 산출물로 깨지지 않는지 확인하기 위한 기준입니다.

## 검수 역할

| 역할 | 관점 | 결과물 |
|---|---|---|
| Reviewer | 50대 전후 ChatGPT·GitHub 입문 독자 | `reports/reader-review.md` |
| Verifier | 기술·출판 무결성 검수자 | `reports/verifier-review.md` |
| Publish Checker | 외부 URL과 EPUB 준비 점검자 | `reports/publish-check.md` |

## Reviewer 검수 기준

Reviewer는 아래 독자처럼 읽습니다.

- 웹서핑과 유튜브 정도만 할 줄 압니다.
- GitHub는 처음 보거나 이름만 들어 보았습니다.
- 코딩 화면이 나오면 불안해합니다.
- 긴 글보다 화면을 보고 따라 하는 방식을 선호합니다.
- 버튼 위치와 다음 행동이 분명해야 계속 진행합니다.

### Reviewer 체크리스트

- 목차만 읽어도 책의 목표가 이해됩니다.
- 첫 사이트 완성이 너무 늦게 나오지 않습니다.
- GitHub 연동 화면에서 불안한 부분을 이미지로 설명합니다.
- 프롬프트 복사 흐름이 이해됩니다.
- 각 이미지 아래 설명이 1~2줄 안에 끝납니다.
- 한 페이지에 행동이 너무 많이 들어가지 않습니다.
- 어려운 용어가 나오면 바로 쉬운 말로 풀어 줍니다.
- ChatGPT 대화창 중심 흐름이 유지됩니다.
- GitHub 웹 UI 직접 조작 중심으로 흐르지 않습니다.
- 타로 이미지 생성 한도 문제를 미리 설명합니다.
- 막혔을 때 다시 부탁할 문장이 충분합니다.

## Verifier 검수 기준

Verifier는 아래 항목을 기계적으로 확인합니다.

- 파일 경로가 계획과 일치합니다.
- 이미지 파일명이 규칙을 따릅니다.
- 깨진 이미지가 없습니다.
- 같은 이미지가 불필요하게 반복되지 않습니다.
- 이미지 번호가 실제 UI 글자보다 약 20% 작습니다.
- 번호가 버튼, 입력칸, 본문 글자를 가리지 않습니다.
- 프롬프트 박스의 3줄 미리보기와 원문 앞 3줄이 일치합니다.
- 복사하기 버튼이 모든 프롬프트 박스에 있습니다.
- 모바일 390px, 430px, 태블릿 768px, 데스크톱 1280px에서 화면이 깨지지 않습니다.
- GitHub Pages 외부 URL에서 결과를 확인했습니다.
- 캐시 무효화 규칙이 들어 있습니다.
- Sigil EPUB 준비 경로가 존재합니다.

## Phase별 검수 루프

| 단계 | Reviewer | Verifier | 산출물 |
|---|---|---|---|
| Phase 1 설계 | 목차와 페이지 흐름 검수 | 이미지·프롬프트·배포 기준 검수 | 설계 보정 |
| Phase 2 원고 | 독자 손 멈춤 지점 검수 | 파일·링크·이미지 참조 검수 | 원고 보정 |
| Phase 3 프롬프트 | 복사해서 쓸 수 있는지 검수 | 원문·미리보기 일치 검수 | 프롬프트 보정 |
| Phase 4 웹북 | 읽기 흐름과 버튼 위치 검수 | 반응형·복사 버튼 검수 | 웹북 보정 |
| Phase 5 EPUB | EPUB 독서 흐름 검수 | Sigil 구조·이미지 경로 검수 | EPUB 준비 보정 |
| Phase 6 외부 확인 | 실제 독자 시나리오 확인 | 외부 URL·캐시 무효화 확인 | 배포 보정 |

```markdown
## 반복되거나 지루한 지점

## 수정 권장 사항

## 통과 여부
```

## 중단 기준

아래 항목이 나오면 다음 Phase로 넘어가지 않습니다.

- 첫 사이트 또는 타로 사이트 결과 화면이 없습니다.
- GitHub 연동 배포 흐름이 이미지 없이 글로만 설명됩니다.
- 프롬프트 전문이 복사할 수 없는 구조입니다.
- 모바일 390px에서 프롬프트 박스가 깨집니다.
- 외부 공개 URL 확인 없이 완료 처리합니다.
- 캐시 무효화 설계가 없습니다.
- Sigil EPUB 대체 경로가 없습니다.

## 완료 기준

- Reviewer가 독자 관점에서 `통과` 또는 `수정 후 통과`로 판단합니다.
- Verifier가 기술·출판 기준에서 `통과` 또는 `수정 후 통과`로 판단합니다.
- Publish Checker가 외부 URL 확인과 캐시 무효화를 확인합니다.

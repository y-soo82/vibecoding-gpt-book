# 바이브코딩 1차 GPT 입문서 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 상세 기획서를 기준으로 바이브코딩 1차 GPT 입문서를 실제 제작 가능한 반응형 전자책 프로젝트로 전환합니다.

**Architecture:** 1차 산출물은 `vibecoding/1/gpt` 아래에 기획·원고·프롬프트·이미지 명세·웹북·EPUB 준비물을 분리해 관리합니다. 먼저 최종 목차와 페이지 설계를 확정하고, 이후 이미지 컷 리스트와 프롬프트 UX를 기준으로 원고·웹북·Sigil 변환 준비·검수 루프를 순차 진행합니다.

**Tech Stack:** Markdown, HTML/CSS/JavaScript 반응형 웹북, Docsify 또는 정적 HTML, GitHub Pages, Sigil EPUB, Playwright 캡처, sharp 이미지 합성, Reviewer/Verifier/Writer/Editor 출판 agents.

## 0. 기준 요약

### 프로젝트 기준

- 프로젝트명: 바이브코딩
- 1차 책: GPT 입문서
- 대상 독자: 웹서핑과 유튜브 정도만 할 줄 아는 비개발자
- 핵심 경험: ChatGPT 대화창만으로 웹사이트를 만들고, ChatGPT GitHub 연동으로 배포합니다.
- 결과물: 웹에서 열리는 사이트 2개
  - 이미지 한 장으로 만든 첫 웹사이트
  - 타로카드 리딩 웹사이트

### 출판 기준

- 반응형 전자책이 1차 상품입니다.
- Sigil EPUB 패키징을 최종 단계에서 고려합니다.
- 전체 분량 목표는 80~100페이지입니다.
- 이미지 비중은 60~70%입니다.
- 긴 프롬프트는 3줄 미리보기만 보여 주고, 우측 상단 복사하기 버튼으로 전문을 복사합니다.
- 이미지 번호는 실제 UI 글자보다 약 20% 작게 배치합니다.
- 반복 이미지와 반복 설명으로 콘텐츠 품질이 낮아지지 않게 합니다.
- 최종 확인은 외부 공개 URL에서 진행합니다.
- 캐시 문제 방지를 위해 noCache, 버전 쿼리, 확인용 URL 쿼리를 사용합니다.

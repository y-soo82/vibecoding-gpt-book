# 출판 감사 수정 리스트

## 1. 요약 판정

현재 공개 웹북은 **출판 불가능** 상태입니다. 외부 URL은 열리지만, 책 본문에 placeholder 문구가 남아 있고, 일부 이미지는 웹북을 다시 캡처한 재귀 이미지이며, ChatGPT/GitHub 핵심 화면이 실제 독자가 따라 할 수 있는 수준의 캡처가 아닙니다.

### 가장 큰 문제 5개

- **P0 캡션 오염**: 이미지 캡션에 “이미지 자리:” 문구가 그대로 노출됩니다.
- **P0 재귀 캡처**: `ch00_03_responsive_book.png`처럼 웹북 화면을 책 이미지로 다시 넣은 컷이 있습니다.
- **P1 핵심 화면 mock 품질**: ChatGPT/GitHub 연결 화면이 실제 UI와 거리가 있어 “보고 따라 하기” 목적을 충족하기 어렵습니다.
- **P1 모바일 이미지 가독성**: 모바일에서는 캡처 안의 글자가 작아 실제 행동 위치를 읽기 어렵습니다.
- **P2 웹북 구조**: 98페이지를 한 페이지에 모두 펼쳐 모바일 스크롤이 지나치게 길고 전자책 페이지 경험이 약합니다.

## 6. 실제 예제 사이트 점검

### 첫 웹사이트

- 외부 URL은 열립니다.
- 이미지 깨짐은 없습니다.
- 모바일에서 화면은 깨지지 않지만, 책 캡처용 결과물로는 핵심 요소가 한 화면에 압축되어 보이지 않습니다.
- 수정 방향: 책에 넣을 캡처는 데스크톱 결과와 모바일 결과를 별도로 촬영하고, 모바일 컷은 제목/이미지/버튼이 한 화면에 들어오도록 캡처 상태를 조정합니다.

### 타로카드 사이트

- 외부 URL은 열립니다.
- 이미지 깨짐은 없습니다.
- 기본 흐름은 보이나, 책 캡처용으로는 각 단계의 핵심 조작 위치가 더 크게 보여야 합니다.
- 수정 방향: 주제 선택, 질문 입력, 카드 3장 선택, 결과, 카드 상세 화면을 각각 확대 가독성 기준으로 다시 캡처합니다.

## 7. 다음 수정 작업 순서

### 가장 먼저 고칠 것

1. 웹북 렌더러 또는 원고에서 “이미지 자리:” 캡션 노출을 제거합니다.
2. 재귀 캡처 이미지와 placeholder처럼 보이는 이미지를 출판용 실제 컷으로 교체합니다.
3. ChatGPT/GitHub 핵심 mock 화면을 실제 UI 구조에 가깝게 다시 제작하거나 민감정보 없는 실제 캡처로 대체합니다.

### 그다음 고칠 것

1. 모바일에서 이미지 내부 글자가 읽히도록 캡처 방식을 세로형/확대형으로 다시 잡습니다.
2. 프롬프트 박스와 캡처 이미지 사이 여백을 줄이고 전자책 페이지 흐름을 정리합니다.
3. 예제 사이트 캡처는 독자가 누를 위치가 보이도록 단계별 컷으로 다시 촬영합니다.

### 나중에 고칠 것

1. 98페이지 전체를 한 화면에 펼치는 웹북 구조를 장별 보기 또는 페이지 단위 보기로 개선합니다.
2. EPUB 변환 전 이미지 유형별 캡션 규칙을 확정합니다.
3. mock 화면을 유지할지 실제 서비스 캡처로 교체할지 최종 편집 정책을 정합니다.

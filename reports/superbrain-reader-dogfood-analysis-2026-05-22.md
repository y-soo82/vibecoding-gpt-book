# 슈퍼브레인 독자 따라하기 문제 분석과 수정안

## 핵심 판단

이번 책은 “설명서”보다 “독자가 같은 결과를 만드는 작업 흐름”에 가까워야 합니다. 따라서 독자가 막히는 지점은 지식 부족보다 준비 누락, 권한 연결, 결과 차이, 이미지 자료 누락에서 발생합니다.

## 문제 분석

### 1. 준비 조건이 본문보다 먼저 보여야 합니다

독자는 Chapter 2에서 바로 이미지를 첨부하고 프롬프트(prompt)를 복사합니다. 이때 ChatGPT 로그인, GitHub 로그인, 사용할 이미지 준비가 먼저 확인되지 않으면 실습이 중간에 멈춥니다.

수정 방향은 Chapter 2 앞부분에 짧은 준비 절을 넣는 것입니다. 준비 절은 길게 설명하지 않고, 로그인 상태와 사용할 이미지만 확인하게 해야 합니다.

### 2. Chapter 3은 한 장처럼 읽혀야 합니다

타로(tarot) 사이트 제작은 가장 긴 실습입니다. 원고 안에 예전 장 번호가 남아 있으면 독자는 “지금 내가 Chapter 3을 읽는 건지, 다른 책 조각을 읽는 건지” 헷갈립니다.

수정 방향은 Chapter 3 안의 큰 흐름을 네 덩어리로 정리하는 것입니다.

- 완성 흐름을 먼저 봅니다
- 카드 이미지 준비 방식을 정합니다
- 화면별로 타로(tarot) 사이트를 만듭니다
- 타로(tarot) 사이트를 웹에 올립니다

### 3. 책 캡처와 실제 결과가 달라질 수 있는 지점을 먼저 관리해야 합니다

책의 핵심 약속은 “기능, 화면 흐름, UI(User Interface), UX(User Experience)가 책 캡처와 최대한 같게 나온다”입니다. 이 약속을 깨는 가장 큰 원인은 카드 이미지 제공 자료 누락과 ChatGPT 결과 변동입니다.

수정 방향은 제공 자료를 사용하는 이유와 이미지 깨짐 대응을 Chapter 3에서 먼저 설명하는 것입니다. 결과가 달라졌을 때는 부록의 재요청 문장으로 연결합니다.

### 4. 프롬프트(prompt) 원문도 책의 일부입니다

프롬프트(prompt) 전문 보기 링크는 독자가 실제로 누를 수 있는 책 요소입니다. 여기에 예전 장 번호가 남아 있으면 책 전체 신뢰가 떨어집니다.

수정 방향은 모든 프롬프트(prompt) 사용 위치를 현재 4챕터 구조에 맞게 갱신하는 것입니다.

### 5. 배포 URL(Uniform Resource Locator) 확인은 작업자가 책임져야 합니다

사용자는 리모트 환경에서 직접 확인하기 어렵다고 했습니다. 따라서 HTML(HyperText Markup Language) 결과물이 만들어지면 Git에 푸시하고 배포 URL(Uniform Resource Locator)을 열어 보는 책임이 작업자에게 있습니다.

수정 방향은 Chapter 2와 Chapter 4에 권한, Pages 반영 시간, 작업자 확인 기준을 넣는 것입니다.

## 이번 수정에서 반영한 내용

- Chapter 2 시작부에 따라하기 전 준비 절을 추가했습니다.
- Chapter 2 배포 절에 권한 화면과 연결 막힘 대응을 추가했습니다.
- Chapter 3 내부 장 번호를 제거하고 4단계 흐름으로 정리했습니다.
- Chapter 3에 제공 자료 누락과 이미지 깨짐 위험을 보강했습니다.
- Chapter 4에 배포 URL(Uniform Resource Locator) 확인 책임을 명확히 넣었습니다.
- 프롬프트(prompt) 원문 사용 위치를 현재 목차 기준으로 갱신했습니다.

## 다음 검수 기준

- 전체 HTML(HyperText Markup Language)에서 깨진 이미지가 없어야 합니다.
- 프롬프트(prompt) 카드는 모두 3줄 미리보기와 복사하기 전문을 가져야 합니다.
- 전체 책에 예전 내부 장 번호가 보이면 안 됩니다.
- Chapter 3은 타로(tarot) 사이트 제작 한 장으로 읽혀야 합니다.
- 실제 ChatGPT 로그인 세션에서 이미지 첨부와 GitHub 연결까지 다시 검수해야 합니다.

## 수정 후 판정

현재 자동 검증 기준에서는 전자책 구조 문제가 해결되었습니다. 다만 ChatGPT 직접 실행은 로그인 세션이 필요한 영역이므로, 다음 검수에서는 사용자의 로그인된 브라우저 또는 승인된 세션에서 이미지 첨부와 GitHub 연결까지 확인해야 합니다.

# 독자 따라하기 검수 진행 기록

## 검수 목적

웹서핑과 유튜브 시청 정도에 익숙한 독자가 책을 처음부터 끝까지 읽고, 책의 프롬프트(prompt)를 따라 하면서 첫 웹사이트와 타로(tarot) 사이트를 만들 수 있는지 확인했습니다.

## 검수 대상

- 전체 책: `webbook/full.html`
- 챕터별 책: `webbook/chapter1.html`, `webbook/chapter2.html`, `webbook/chapter3.html`, `webbook/chapter4.html`
- 첫 웹사이트 예제: `examples/first-site/index.html`
- 타로(tarot) 사이트 예제: `examples/tarot-site/index.html`
- 프롬프트(prompt) 원문: `prompts/*.md`

## 직접 따라하기 진행 상태

| 구간 | 독자 행동 | 확인 결과 |
| --- | --- | --- |
| Chapter 1 | AI(Artificial Intelligence), LLM(Large Language Model), 바이브코딩(vibe coding) 개요 읽기 | 책의 목표와 독자 수준은 확인됩니다. |
| Chapter 2 | 이미지 첨부 후 첫 웹사이트 제작 프롬프트(prompt) 복사 | 프롬프트(prompt)는 3줄 미리보기와 복사하기 전문 구조로 제공됩니다. |
| Chapter 2 | 첫 웹사이트 결과 확인 | 로컬 예제 사이트가 열리고 대표 이미지가 정상 표시됩니다. |
| Chapter 2 | GitHub 연결과 배포 흐름 확인 | GitHub 연결 권한 화면과 배포 URL(Uniform Resource Locator) 확인 흐름이 보입니다. |
| Chapter 3 | 타로(tarot) 사이트 완성 흐름 확인 | 메인, 주제, 질문, 카드 뽑기, 결과, 배포 흐름을 한 장 안에서 따라갑니다. |
| Chapter 3 | 타로(tarot) 예제 사이트 확인 | 예제 사이트가 열리고 시작 버튼과 주제 선택, 카드 선택 구조가 보입니다. |
| Chapter 4 | 막혔을 때 재요청 문장 확인 | 잘못된 답변, 배포 주소, 이미지 깨짐, 다음 아이디어 대응 문장이 있습니다. |

## ChatGPT 직접 입력 시도

Playwright로 `https://chatgpt.com/` 접속을 시도했습니다.

- 접속 결과: ChatGPT 첫 화면 접근 성공
- 로그인 상태: 로그인 필요 화면으로 확인
- 입력창 감지: 입력 요소는 보이나, 이미지 첨부와 GitHub 연결 검증은 로그인 세션이 필요합니다.
- 결론: 이번 자동 검수에서는 실제 이미지 첨부, ChatGPT 결과 생성, GitHub 연결 배포까지 끝까지 실행하지 못했습니다.

이 제한은 책의 문제라기보다 검수 환경의 로그인 세션 문제입니다. 다만 일반 독자도 로그인과 권한 연결에서 막힐 수 있으므로, 전자책에는 따라하기 전 준비와 권한 화면에서 막혔을 때의 안내를 보강해야 합니다.

## 발견한 문제

| 번호 | 문제 | 독자에게 생기는 영향 | 수정 방향 |
| --- | --- | --- | --- |
| 1 | Chapter 2 시작 전 로그인, 이미지 준비, GitHub 준비 조건이 짧습니다. | 독자가 프롬프트(prompt)를 보내기 전에 무엇을 준비해야 하는지 놓칠 수 있습니다. | Chapter 2에 따라하기 전 준비 절을 추가합니다. |
| 2 | Chapter 3 원고 안에 예전 내부 장 번호 흐름이 남아 있습니다. | 4챕터 책인데 중간에 다른 장처럼 느껴질 수 있습니다. | Chapter 3 하나의 흐름으로 재정리합니다. |
| 3 | 타로(tarot) 카드 제공 자료가 빠졌을 때 결과가 달라지는 위험이 충분히 강조되지 않았습니다. | 책 캡처와 실제 결과의 UI(User Interface), UX(User Experience)가 달라질 수 있습니다. | 제공 자료 확인과 깨짐 대응 문장을 보강합니다. |
| 4 | 프롬프트(prompt) 전문 보기 화면의 사용 위치가 예전 장 번호를 가리킵니다. | 독자가 원문을 눌렀을 때 현재 목차와 맞지 않는 정보를 봅니다. | 프롬프트(prompt) 사용 위치를 현재 4챕터 구조로 갱신합니다. |
| 5 | 배포 URL(Uniform Resource Locator)이 안 열릴 때 작업자 확인 기준이 약합니다. | 리모트 환경에서 사용자가 직접 확인하지 못할 때 막힙니다. | 부록에 배포 URL 확인 책임과 확인 기준을 추가합니다. |

## 수정 반영 내역

- Chapter 2에 `따라하기 전에 준비합니다` 절을 추가했습니다.
- Chapter 2 배포 절에 권한 화면과 GitHub Pages 막힘 대응 문장을 추가했습니다.
- Chapter 3을 `타로앱을 만듭니다` 한 장 흐름으로 재정리했습니다.
- Chapter 3의 예전 내부 장 번호와 제작 단계 문구를 제거했습니다.
- Chapter 3에 제공 자료가 빠질 때 결과가 달라지는 기준을 보강했습니다.
- Chapter 4에 배포 URL을 작업자가 열어 보고 보고해야 한다는 기준을 보강했습니다.
- 프롬프트(prompt) 원문의 사용 위치를 현재 4챕터 구조에 맞게 갱신했습니다.

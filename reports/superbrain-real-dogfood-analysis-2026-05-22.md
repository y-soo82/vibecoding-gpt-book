# 슈퍼브레인 실제 dogfood 문제 분석

작성일: 2026-05-22

## 핵심 문제

현재 전자책은 독자가 그대로 따라 하면 같은 결과가 나온다는 약속을 아직 완전히 충족하지 못합니다. 기존 핵심 이미지와 진행 기록은 실제 ChatGPT 실행 기반이 아니었고, 2026-05-22 재실행으로 Chapter 2 첫 웹사이트 생성 일부만 실제 증거로 확보했습니다.

## 문제점

1. 실제 ChatGPT 입력 기록이 부분적으로만 생겼습니다.
   - Chapter 2의 첫 웹사이트 제작 프롬프트(prompt)는 실제 ChatGPT에 입력했습니다.
   - 버블 공연 사진도 실제 ChatGPT 탭에 첨부했습니다.
   - 실제 생성 HTML, 데스크톱 결과, 모바일 결과를 확보했습니다.
   - Chapter 3의 타로(tarot) 프롬프트(prompt)를 실제 ChatGPT에 넣은 기록이 없습니다.

2. 실제 GitHub 화면 캡처가 없습니다.
   - GitHub 홈, 권한 승인, ChatGPT 커넥터, 배포 URL 답변 화면은 실제 웹 화면으로 확정할 수 없습니다.
   - Chapter 2 첫 웹사이트의 GitHub Pages 배포도 아직 진행하지 않았습니다.

3. 타로(tarot) 결과는 실제 GPT 결과물이 아닙니다.
   - 현재 화면은 로컬 예제 또는 출판용 화면이므로, 독자가 GPT로 만든 결과와 같다고 보장할 수 없습니다.

4. 기존 dogfood 문서는 완료 근거가 아닙니다.
   - 기존 문서는 로컬 구조 검수이며, 실제 독자 실행 기록으로 부를 수 없습니다.

5. ChatGPT 파일 다운로드 흐름은 추가 보정이 필요합니다.
   - `index.html 바로 보기/다운로드`는 Chrome에서 `ERR_BLOCKED_BY_CLIENT`로 차단됐습니다.
   - 직접 URL 다운로드도 HTTP 403으로 실패했습니다.
   - 이번에는 코드 보기 탭의 복사 기능으로 실제 HTML 원문을 확보했습니다.
   - 독자용 책에는 “다운로드가 막히면 코드 보기에서 복사한다”는 흐름을 바로 넣기보다, ChatGPT UI에서 가장 안정적인 파일 확보 흐름을 추가 검증한 뒤 반영해야 합니다.

## 수정 방향

1. Chapter 2 첫 웹사이트 생성 증거는 실제 캡처로 교체할 수 있습니다.
2. Chapter 2 본문에는 파일 업로드 권한, 이미지 첨부 성공, 답변 생성, 미리보기 확인, 코드 보기 복사 흐름을 실제 결과 기준으로 다시 써야 합니다.
3. GitHub 연결과 배포는 실제 권한 화면과 최종 URL을 기준으로 캡처합니다.
4. Chapter 3 타로(tarot) 제작은 org 기준 기능과 UI(User Interface), UX(User Experience)를 기준으로 실제 GPT 결과를 유도합니다.
5. 실제 GPT 결과가 책 기획과 다르면 프롬프트(prompt)를 먼저 고치고, 그다음 전자책을 수정합니다.

## 전자책 수정 원칙

- 실제 캡처가 없는 장면은 실제 캡처처럼 설명하지 않습니다.
- 예제 사이트 화면은 예제 사이트 화면으로만 표시합니다.
- 시뮬레이션 화면은 핵심 실습 증거로 쓰지 않습니다.
- 실제 따라하기가 막힌 지점을 부록이나 본문 안내에 반영합니다.

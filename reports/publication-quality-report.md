# 출판 품질 점검 보고서

## 점검 범위

- `manuscript/` 원고 10개
- `prompts/` 프롬프트 원문 7개
- `webbook/` 반응형 웹북
- `webbook/images/` 출판용 이미지 98개
- `examples/` 예제 사이트 2개
- GitHub Pages 공개 URL

## 원고 점검 결과

- 원고는 98개 페이지 자리로 구성되어 80~100페이지 목표에 맞습니다.
- 각 페이지는 이미지 자리, 짧은 설명, 독자 행동 안내를 중심으로 구성되어 있습니다.
- 프롬프트 전문은 본문에 길게 들어가지 않고 프롬프트 박스 자리로 분리되어 있습니다.
- GitHub 배포 설명은 GitHub 웹 UI 직접 조작보다 ChatGPT GitHub 연동 대화 배포 흐름을 중심으로 유지되어 있습니다.
- 일부 화면은 실제 서비스 UI가 아니라 출판용 mock 캡처를 사용하므로, 최종 편집 때 캡션에서 `출판용 예시 화면` 의미를 유지해야 합니다.

## 웹북 점검 결과

- 웹북은 10개 장을 목차로 접근할 수 있습니다.
- 프롬프트 박스 24개는 3줄 미리보기, 복사하기 버튼, 전체 보기 토글을 제공합니다.
- 이미지 98개가 모두 실제 PNG로 표시됩니다.
- placeholder는 0개입니다.
- noCache 메타 태그와 버전 쿼리를 유지했습니다.
- 최신 웹북 빌드 번호는 `20260516-pub-001`입니다.

## 이미지 점검 결과

- 전체 이미지 자리: 98개
- 실제 생성 이미지: 98개
- 남은 이미지: 0개
- Playwright 직접 생성 원칙을 지켰습니다.
- ChatGPT/GitHub 계정 화면은 실제 계정 정보 노출을 피하기 위해 출판용 mock 화면으로 대체했습니다.
- mock 대체 내역은 `reports/phase8-9-image-completion-report.md`에 기록했습니다.

## 프롬프트 UX 점검 결과

- 본문에는 프롬프트 전문이 아니라 3줄 미리보기만 보입니다.
- `복사하기` 버튼을 누르면 전문이 클립보드에 복사됩니다.
- 복사 성공 시 버튼 문구가 `복사됨`으로 바뀝니다.
- `전체 보기` 토글로 전문 확인이 가능합니다.
- EPUB에서는 복사 버튼 제한 가능성이 있으므로 `epub/sigil-epub-preflight.md`의 프롬프트 전문 부록 방식을 사용합니다.

## 출판 전 남은 주의점

- mock 화면은 실제 ChatGPT/GitHub UI와 완전히 같다는 의미가 아닙니다.
- 실제 서비스 화면을 쓰려면 별도 캡처용 계정과 마스킹 기준을 먼저 확정해야 합니다.
- Sigil 변환 후에는 리더 앱별 이미지 크기와 프롬프트 전문 링크를 다시 확인해야 합니다.
